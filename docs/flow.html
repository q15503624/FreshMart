🛒 FreshMart 簡易下單系統

無會員、無金流，以「手機號碼 + 查詢碼 + 轉帳末五碼」完成訂單確認的下單網站。下單 → 轉帳回報 → 管理者核帳 → 出貨 → 查詢進度。

使用者操作 系統自動 管理者操作

整體流程閉環

使用者
1. 瀏覽商品並下單
瀏覽商品、加入購物車，結帳時填寫姓名、手機號碼、收件地址（LINE ID / 備註可選）。
↓
系統
2. 建立訂單，產生編號與查詢碼
產生訂單編號、查詢碼與付款資訊（金額、銀行、帳號）。
待付款
↓
使用者
3. 自行轉帳並回報付款
完成轉帳後，從「我已付款」連結或用「手機 + 查詢碼」進入回報頁，填寫轉帳末五碼、金額、時間。
已回報付款
↓
系統
4. 即時通知管理者
透過 Telegram Bot 通知（Email 備援），含訂單與回報明細。金額不符會特別標示異常。
↓
管理者
5. 後台核帳
核對銀行入帳資料，可操作「確認收款」「付款異常」或「取消訂單」。
已確認收款
↓
管理者
6. 備貨、出貨並更新狀態
出貨後在後台更新出貨狀態、出貨時間、物流方式（物流單號 / 備註可選）。
已出貨
↓
使用者
7. 查詢訂單進度
用手機號碼 + 查詢碼查詢付款狀態、出貨狀態、物流資訊與更新時間。

訂單狀態流程

正常流程：

待付款 → 已回報付款 → 已確認收款 → 備貨中 → 已出貨

異常流程：

待付款→已取消 已回報付款→付款異常 付款異常→已確認收款 / 已取消

付款回報的兩種入口

方式一：訂單完成頁「我已付款」

  • 下單成功後直接顯示連結
  • 連結可帶入訂單資訊
  • /payment-report?orderNo=...&code=...

方式二：手機號碼 + 查詢碼

  • 適用已離開完成頁的情況
  • 從「訂單查詢 / 我已付款」進入
  • 查到訂單後再點「我已付款」

付款回報通知範例

💰 有新的付款回報 訂單編號：F202605300001 客戶姓名：王小明 手機號碼：0912***789 訂單金額：$1,280 回報金額：$1,280 轉帳末五碼：12345 轉帳時間：2026-05-30 15:32 目前狀態：已回報付款，待核帳 後台查看：https://your-domain.com/admin/orders/F202605300001
