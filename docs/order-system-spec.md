# FreshMart 簡易下單系統 — 需求規格文件

> 版本：v0.1（雛形，待優化）
> 最後更新：2026-05-30

---

## 0. 一句話總覽

這個系統是一個**無會員、無金流、以轉帳回報為核心**的簡易下單網站。

使用者下單後取得**訂單編號**與**查詢碼**，付款後回網站填寫**轉帳末五碼**回報付款；系統即時通知管理者，管理者核帳後出貨，並在後台更新狀態，使用者可用**手機號碼 + 查詢碼**查詢訂單進度。

### 核心目標

讓使用者可以在網站上完成下單、自行轉帳、回報付款資訊；管理者收到通知後確認款項並安排出貨，最後同步更新訂單狀態讓使用者查詢。

### 設計原則

- 不做複雜會員系統。
- 不串接金流。
- 改用「**手機號碼 + 查詢碼 + 轉帳末五碼**」完成訂單確認流程。

---

## 1. 使用者下單流程

1. 使用者進入網站後，可瀏覽商品並加入購物車。
2. 結帳時填寫基本訂購資料。
3. 送出訂單後，系統建立一筆訂單並產生訂單編號、查詢碼與付款資訊。

### 1.1 結帳填寫欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 姓名 | ✅ | 訂購人姓名 |
| 手機號碼 | ✅ | 用於後續查詢與通知 |
| 收件地址 | ✅ | 出貨地址 |
| LINE ID | ⬜ | 可選 |
| 備註 | ⬜ | 可選 |

### 1.2 系統產生內容

- 訂單編號
- 訂單查詢碼
- 付款資訊（金額、銀行、帳號）

範例：

```
訂單編號：F202605300001
查詢碼：728391
應付金額：$1,280
轉帳銀行：XXX銀行
轉帳帳號：XXX-XXXX-XXXX
```

> 此時訂單狀態為：**待付款**

---

## 2. 付款回報方式

使用者完成轉帳後，需回到網站回報付款。提供兩種入口。

### 方式一：訂單完成頁的「我已付款」連結

- 下單成功後，訂單完成頁直接顯示「**我已付款**」。
- 點擊後進入付款回報頁，連結可帶入訂單資訊：

```
/payment-report?orderNo=F202605300001&code=728391
```

### 方式二：手機號碼 + 查詢碼

- 若使用者已離開訂單完成頁，可從「**訂單查詢 / 我已付款**」頁面進入。
- 輸入**手機號碼**與**訂單查詢碼**。
- 系統查到訂單後顯示：訂單編號、訂購人、訂單金額、目前狀態。
- 點擊「**我已付款**」後進入回報表單。

### 2.1 付款回報表單欄位

| 欄位 | 必填 | 說明 |
|------|------|------|
| 轉帳末五碼 | ✅ | 對帳關鍵 |
| 轉帳金額 | ✅ | 與訂單金額比對 |
| 轉帳時間 | ✅ | |
| 轉帳人姓名 | ⬜ | 可選 |
| 備註 | ⬜ | 可選 |

> 送出後，訂單狀態更新為：**已回報付款**

---

## 3. 系統通知管理者

使用者送出付款回報後，系統需**即時通知**管理者。

第一版建議：

- **Telegram Bot 通知**（主要）
- **Email 備援通知**

### 3.1 通知內容範例

```
💰 有新的付款回報

訂單編號：F202605300001
客戶姓名：王小明
手機號碼：0912***789

訂單金額：$1,280
回報金額：$1,280
轉帳末五碼：12345
轉帳時間：2026-05-30 15:32

目前狀態：已回報付款，待核帳

後台查看：
https://your-domain.com/admin/orders/F202605300001
```

### 3.2 金額異常標示

當回報金額與訂單金額不一致時，通知需特別標示：

```
⚠️ 付款金額異常
訂單金額：$1,280
回報金額：$1,200
差額：-$80
```

## 4. 後台核帳與發貨流程

管理者收到通知後進入後台查看訂單。

### 4.1 後台訂單列表欄位（至少）

- 訂單時間
- 訂單編號
- 客戶姓名
- 手機號碼
- 訂單金額
- 轉帳末五碼
- 付款回報時間
- 訂單狀態
- 操作按鈕

### 4.2 核帳操作

管理者核對銀行入帳資料後可操作：

- **確認收款** → 狀態變為「已確認收款」
- **付款異常** → 狀態變為「付款異常」
- **取消訂單** → 狀態變為「已取消」

### 4.3 出貨更新欄位

確認收款後備貨、出貨，出貨後在後台更新：

| 欄位 | 必填 | 說明 |
|------|------|------|
| 出貨狀態 | ✅ | |
| 出貨時間 | ✅ | |
| 物流方式 | ✅ | |
| 物流單號 | ⬜ | 可選 |
| 備註 | ⬜ | 可選 |

> 更新後，訂單狀態變成：**已出貨**

---

## 5. 使用者查詢訂單狀態

使用者回到訂單查詢頁，輸入**手機號碼**與**訂單查詢碼**查詢。

可看到：

- 訂單編號
- 訂單金額
- 付款狀態
- 出貨狀態
- 物流資訊
- 訂單更新時間

---

## 6. 訂單狀態機

### 6.1 狀態清單

| 狀態 | 說明 |
|------|------|
| 待付款 | 訂單建立後初始狀態 |
| 已回報付款 | 使用者已回報轉帳資訊 |
| 已確認收款 | 管理者核帳通過 |
| 備貨中 | 準備出貨 |
| 已出貨 | 已交付物流 |
| 已取消 | 訂單取消 |
| 付款異常 | 金額不符或對帳失敗 |

### 6.2 正常流程

```
待付款 → 已回報付款 → 已確認收款 → 備貨中 → 已出貨
```

### 6.3 異常流程

```
待付款        → 已取消
已回報付款    → 付款異常
付款異常      → 已確認收款
付款異常      → 已取消
```

### 6.4 系統閉環

```
使用者下單
  ↓
填寫姓名、手機、地址
  ↓
系統產生訂單編號與查詢碼
  ↓
使用者自行轉帳
  ↓
使用者點擊「我已付款」
  ↓
填寫轉帳末五碼、金額、時間
  ↓
系統通知管理者
  ↓
管理者後台核帳
  ↓
確認收款
  ↓
安排出貨
  ↓
後台更新出貨狀態
  ↓
使用者用手機 + 查詢碼查詢訂單狀態
```
